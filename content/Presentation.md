## 段落三：EQS 全球報價系統（約 3.5 分鐘）

**背景與我的角色**
EQS 是東元機電的全球機電產品報價平台 2.0。
我在這個專案擔任 SD（系統設計）+ UIUX，是我第一次主導一個系統級別的專案。

**問題在哪**
舊系統有兩個核心問題：
第一：業務報價有 100 多個表單欄位，沒有卡控機制。
填錯了、填漏了，都要等到進簽核流程才被退回，流程要重新跑，耗時又耗人力。

第二：架構用多個獨立區塊 + ng-if 切換，
業務每次切換視圖，DOM 就銷毀重建，資料重置、API 重打——測試發現「操作中斷率 100%」。

**我怎麼解**
我做了兩件事：

第一，流程卡控設計：
依不同產品類型設定不同欄位規則——必填、格式限制、數量與加價項目的綁定邏輯。
在送出前，如果金額低於企業要望價，強制要求填寫原因才能往下走。
然後把簽核流程狀態視覺化，讓主管一眼知道流程卡在哪一個人身上。

第二，架構 A/B 測試：
我提出用「共用主容器 + 權限模型控制顯示層」來取代多區塊 ng-if。
結果是：DOM 不再銷毀、狀態集中管理、跨權限切換操作流暢度大幅提升。

**成果**
企業對報價的內控被實際強化，不再靠業務個人經驗。
跨地域團隊有了統一的介面規格與設計 Token 基礎，降低溝通與實作偏差。

## 段落四：MES 智慧製造系統（約 4 分鐘）

**背景與我的角色**
MES 是東元集團的智慧製造管理系統，我主導前端與設計，協同三名前端開發人員，完整走過系統五個階段的生命週期。

**問題在哪**
製造現場的核心問題：工時用紙本抄寫、人員上下工記錄分離於工單系統、設備狀態無法即時綁定。
結果是：數據可信度低、生產異常無法即時追蹤、人力與設備稼動率難以準確衡量。

**我怎麼解**

第一塊：即時報工系統（PC + PDA 雙端）

PC 端：透過平行作業管理機制，維持同一工作站多任務、多角色切換的作業連續性。人員、設備、工單三方即時綁定，工時動態累積與標準時映射。

PDA 端：我為工廠現場環境做了特別設計——
掃描行為取代手動輸入，把下工與完工設為核心快速 CTA，按鈕大小與色彩對比為工廠環境優化。
**成果：上工操作時間從 1 分 20 秒縮短至 20 秒，效率提升 75%。**

第二塊：即時監控平台（戰情室）

這是讓管理層「一個頁面掌握全廠即時狀態」的看板，整合五個數據維度：
- 全球接單未出貨分佈（地圖標點）
- 誤期客戶工號台數（橫向 bar 排名）
- 各產線缺料工號台數
- 周製程負載率（Gauge 圖）
- 年度各產線準交率（折線柱狀混合圖）

數據處理方式：用 forkJoin 並行打兩支 API 縮短等待時間，準交率計算邏輯（實際÷計畫×100）統一在 service 層，Timer 定時刷新頻率由 config 控制——不改 code 就能調整更新頻率。

地圖節點 hover 才去打那個區域的 API——這是一個「使用者操作行為 = 數位軌跡信號」的設計，系統根據操作者的關注點給出對應數據，而不是一次性全部載入。

## 段落五：Nexus 企業級 UI 元件庫（約 2 分鐘）

在做 EQS 跟 MES 的過程中，我發現一個問題：
設計到前端這段的落差，在每個專案都要重新手動設定，過程非常繁瑣、成本非常高。

所以我自己動手解決這個問題——建了一套 Angular 企業元件庫：Nexus。

**核心鏈路：**
Figma（設計語言定義）→ JSON（結構轉譯）→ CSS Token（實作映射）→ Angular Component（落地應用）

這個鏈路的關鍵是打通了「設計與工程之間的治理斷層」——
設計師改了 Figma Variable，工程端不需要手動同步，Token 自動更新。

**AI Agent 的應用：**
我用 AI Agent 讀取 Figma 的 node-id，對應元件庫的元件名稱，自動拼裝 UI Block。
同時用 AI 把 React 元件源碼自動轉成 Angular，60% 的元件由此生成。

**CLI 工具：**
`npx @nexus/cli add button`——元件源碼直接複製進你的專案，沒有版本鎖定風險，完全可客製。

**資安整合：**
CI/CD 整合 SAST 白箱資安工具，自動阻擋高風險漏洞——前端自主成為安全防線的一環。

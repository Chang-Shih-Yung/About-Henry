## 來到金融業的動機

雖然我先前的專案一個是工廠端、一個是業務端，但過程中能夠發現一些相似之處，比如：**高風險流程中，數據有沒有被正確轉化成決策依據。**

像剛剛展示的看板，可以很明顯的展示：生產數據有了，但沒有整合、沒有卡控，管理者依然無法一目瞭然。
金融業是：客戶的數位行為數據有了，但路徑沒被分析、導引沒被設計，轉換就流失了。

我有實際把這件事做通的經驗——從原始資料整合、到流程卡控、到視覺化呈現。
我認為這套能力可以直接應用在金融場域，這是我來的原因。

## 最有成效的兩個案子

**第一個：多權限卡控 × 防呆機制（在全球報價系統跟智慧製造系統都有應用）**
原本業務報價時有 100 多個表單要填寫，過程中沒有明確卡控或提示（填寫過程非常吃個人經驗）——導致業務往下送到簽核流程又被退回、流程又要重新跑。
我協同 SA 跟後端，重新設計卡控流程：
- 依不同產品加價項目提供不同卡控，比如什麼欄位必填、單選多選、只能填數字、字數限制等等。
- 在送出前，如果金額低於企業要望價會跳出原因欄位填寫，否則不能往下走
- 以及把簽核流程狀態視覺化，讓高階主管一眼看出流程卡在哪

→ 企業對報價的內控實際被強化，不再靠業務個人經驗。減少流程重來的時間耗損，與新進業務的操作負擔

**第二個：AI Agent 自動化 × Figma 到前端（Nexus 元件庫）**
我建了一套 Angular 企業元件庫，並自己串 AI Agent 做自動化維護：
- Figma Variables → JSON → CSS Token 


→ Angular Component 流程全打通

- 用 AI Agent MCP 查找 Figma section 的 node-id，然後回頭對應查找元件庫的元件名稱，最後可以拼接figma上所畫的UI Block。
- CLI 一鍵安裝：`npx @nexus/cli add button`，源碼直接進專案
- 整合資安白箱工具進 CI/CD，自動阻擋高風險漏洞

→ 目標是：提升設計到前端這段的效率，因為我以往的經驗這些部分都要by project手動設定，過程非常繁瑣。

## 遇到的困難與解法

**困難一：客戶說不清需求**
→ 不只聽，直接觀察操作行為，把使用者口中的「點來點去」，轉成系統內的流程節點，找出真正的斷點。比如：MES專案，將報工流程的必填欄位整合預帶資訊＋快速紀錄查找，讓他們可以快速從歷史報工紀錄完成報工。

**困難二：舊系統不能停機重構**
→ 梳理核心流程後分階段替換，舊新系統平行運作，不中斷現場作業。比如後端有增加資料欄位，那我在前端除了要先做預告、教育人練，在迭代過程中也要分權重去更新區塊。不能一次改太多，會讓既有使用者疑惑。

**困難三：流程沒有卡控，全靠人盯**
→ 前端先卡（欄位條件限制）+ 協同後端制定簽核層級，讓系統替代人的經驗法則。也降低教育成本以及版本迭代。

## 富邦 + App 觀察

**優點與競品比較**
- 首頁卡牌規劃，功能集中。整體 App 架構比國泰 CUBE 更輕，國泰功能多但入口太深，功能多卻找不到是一大痛點，使用者在入口就會先疑惑。
- 比起台新銀行：我認為轉帳流程相對順暢，步驟線性化做得不錯。台新會再轉帳後才莫名疊加狀態，有點干擾後續行為


**使用者實際痛點觀察**
- 有看到其他user説：Web 功能比 App 完整，這是現實，但我認為 App 本來就不是要複製 Web——而是給外出快速操作用的，這個取捨我覺得是對的

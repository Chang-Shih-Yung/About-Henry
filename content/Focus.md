## 最有成效的兩個案子

**第一個：多權限卡控 × 防呆機制（在全球報價系統跟智慧製造系統都有應用）**
原本業務報價 100 多個表單，完全沒有卡控——任何條件都能送出，導致往下送到簽核流程又被退回、流程又要重新跑。
我協同 SA 跟後端，重新設計卡控流程：
- 依不同產品加價項目提供不同卡控
- 金額低於要望價會跳出原因欄位填寫，否則不能往下走
- 簽核流程狀態視覺化，高階主管一眼看出流程卡在哪

→ 企業對報價的內控實際被強化，不再靠業務個人經驗。

**第二個：AI Agent 自動化 × Figma 到前端（Nexus 元件庫）**
我建了一套 Angular 企業元件庫，並自己串 AI Agent 做自動化維護：
- Figma Variables → JSON → CSS Token 

→ Angular Component 全鏈路打通

- 用 AI Agent 把 React 元件自動轉成 Angular，60% 元件由此生成
- CLI 一鍵安裝：`npx @nexus/cli add button`，源碼直接進專案
- 整合資安白箱工具進 CI/CD，自動阻擋高風險漏洞

→ 元件開發時間降低 90%+，設計到交付的流程完全打通

## 遇到的困難與解法

**困難一：客戶說不清需求**
→ 不只聽，直接觀察操作行為，把「點來點去」轉成流程節點，找出真正的斷點

**困難二：舊系統不能停機重構**
→ 梳理核心流程後分階段替換，舊新系統平行運作，不中斷現場作業

**困難三：流程沒有卡控，全靠人盯**
→ 前端先卡（欄位條件限制）+ 簽核層級設計，讓系統替代人的自律

## 富邦 + App 觀察

**優點與競品比較**
- 首頁卡牌規劃，功能集中。整體 App 架構比國泰 CUBE 更輕，國泰功能多但入口太深，功能多卻找不到是一大痛點，使用者在入口就會先疑惑。
- 比起台新銀行：我認為轉帳流程相對順暢，步驟線性化做得不錯。台新會再轉帳後才莫名疊加狀態，有點干擾後續行為


**使用者實際痛點觀察**
- 有看到其他user説：Web 功能比 App 完整，這是現實，但我認為 App 本來就不是要複製 Web——而是給外出快速操作用的，這個取捨我覺得是對的
